30天練習加深了對html, CSS, javascript 的理解，根基亦更扎實。

大多數javascript 應用都是分作三個步驟:

1. 抓取html elements, 例如: const button = document.querySelector('.my-button');

2. 加入eventListener, 例如: button.addEventListener('click', handleClick);

3. 完成handle function, 例如: handleClick() { console.log('Hello world');}

此外，browser提供了很多的Web API, 多查MDN學習套用各種API亦很有幫助。

Day 14: Demo : Shallow copy, deep copy 

類似pointer的概念
以下方法new object/array 會指向original object/array 的 memory address, 修改新的會同時影響原來的
const players = ['Wes', 'Sarah', 'Ryan', 'Poppy'];
const team = players;

解決方法: deep copy
const team2 = players.slice();
const team3 = [].concat(players);
const team4 = [...players];
const team5 = Array.from(players);
然而以上方法只能做一層deep copy, 如果object 裡有object/ array 方法會失效
可以將obj換成json再換回來, 只限json obj, method/function 會失效
var obj2 = JSON.parse(JSON.stringify(obj1));
或使用jquery $.extend or lodash library

//-ms代表【ie】內核識別碼
//-moz代表火狐【firefox】內核識別碼
//-webkit代表谷歌【chrome】/蘋果【safari】內核識別碼
//-o代表歐朋【opera】內核識別碼
為什麼要加識別碼：
在標準尚未確定時，部分瀏覽器已經根據最初預定實現了部分功能，為了與隨後確定下來的標准進行兼容，
因此某些瀏覽器使用了自己的專有性與標准進行區分，當標準糾正後， 各大瀏覽器將逐步支持不帶初始的css3新屬性。
目前已有很多私有共識可以不寫了，但為了兼容老版本的瀏覽器，可以繼續沿用專有的和標準方法，逐漸過渡。
參考：https：//www.cnblogs.com/EasonJim/p/6140097.html

e.isTrusted : 防作弊
Event 介面的 isTrusted 唯讀屬性為一個布林值，
若事件物件是由使用者操作而產生，則 isTrusted 值為 true。
若事件物件是由程式碼所建立、修改，或是透過 EventTarget.dispatchEvent() 來觸發，則 isTrusted 值為 false。
參考: https://developer.mozilla.org/zh-TW/docs/Web/API/Event/isTrusted
